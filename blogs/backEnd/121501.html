自动部署---【CICD】

# 1. 手动编译

本地压缩--打包的文件夹

用xftp上传至研发服务器

# 现将本地打包的文件夹解缩
unzip dist.zip 

# docker tag  可以docker images 查看照搬即可 --标签
docker tag 10.254.9.21/library/XXXX_NAME 10.254.9.21/library/XXXX_NAME:9.25 

# 从Dockerfile构建镜像：docker build  -t ImageName:TagName dir
# . 表示当前目录，即Dockerfile所在目录
docker build -t 10.254.9.21/library/XXXX_NAME:12.16 -f Dockerfile .  看清了一个空格一个点

# 将本地的镜像上传到镜像仓库：docker push  NAME[:TAG]
docker push 10.254.9.21/library/XXXX_NAME:9.25

# 查找出XXXX_NAME镜像
docker images|grep XXXX_NAME

# docker save 导出：  将指定镜像保存成 tar 归档文件： 
# >表示输出到文件，10.254.9.21/library/XXXX_NAME:9.25为目标文件，
#  XXXX_NAME:20.08.05.10.tar.gz是源镜像名
docker save 10.254.9.21/library/XXXX_NAME:9.25 >  XXXX_NAME:20.08.05.10.tar.gz

# 2. 手动部署

将编译之后的镜像 上传至 部署服务器

去想要部署的环境部署-----转到部署服务器

# docker load : 导入使用 docker save 命令导出的镜像
docker load -i XXXX_NAME.11.13.01.tar.gz

# 换个tag
docker tag registry.paas/XYYX/XXXX_NAME:11.13.01 registry.paas/XYYX/XXXX_NAME-vue:20.11.13.01

# 修改 yaml文件
vi ops-vue.yaml
-----------------------------------------------------
containers：下面 修改  
   image： registry.paas/XYYX/XXXX_NAME-vue:20.11.13.01
-----------------------------------------------------

# 停止现在的kubectl进程
kubectl delete -f ops-vue.yaml

# 创建新的kubectl进程
kubectl create -f ops-vue.yaml

# 列出所有运行的Pod信息
kubectl get pod | grep bc-deep
